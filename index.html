LENGUAJES DE PROGRAMACION

Un lenguaje de programación es un lenguaje formal, es decir, un lenguaje con reglas gramaticales bien definidas, que le proporciona a un programador, la capacidad de programar una serie de instrucciones o secuencias de órdenes en forma de algoritmo con el fin de controlar el comportamiento físico o lógico de un sistema informático, de manera que se puedan obtener diversas clases de datos o ejecutar determinadas tareas.

Diferencia entre un lenguaje compilado e interpretado:

  • Compilado: En un lenguaje compilado, la máquina de destino traduce directamente el programa. Las instrucciones vienen enviadas directamente al procesador. Un lenguaje compilado genera un file binario no modificabl

  • Interpretado: En un lenguaje interpretado, el código fuente no es directamente traducido por la máquina de destino. Las instrucciones deben traducirse antes de llegar al procesador. El programa debe traducirse cada vez aumentando el tiempo de ejecución.

¿QUÉ ES UN COMPILADOR?

Un compilador es un programa que traduce código fuente escrito en un lenguaje de alto nivel como Java, a un lenguaje legible por la máquina llamado código objeto, lenguaje de destino o incluso lenguaje ensamblador, además, informa de errores al leer el código.


TIPOS DE COMPILADORES
  • Compiladores de una sola pasada: Generan el código máquina a partir de una única lectura del código fuente

  • Compiladores de varias pasadas: necesitan leer el código fuente varias veces antes de poder producir el código máquina.

  • Compiladores cruzados: crean código para una plataforma diferente a la que se está utilizando para compilar el código original. Por ejemplo, se puede usar un compilador cruzado para generar código para una plataforma de 64 bits en una computadora de 32 bits.

  • Compiladores JIT (Just-in-Time): generan código de máquina en tiempo de ejecución en lugar de generarlo antes de ejecutar el programa. Esto puede mejorar el rendimiento al permitir que el compilador optimice el código específico para la máquina en la que se está ejecutando el programa.

  • Compiladores incrementales: generan un código objeto instrucción por instrucción (en vez de hacerlo para todo el programa) cuando el usuario teclea cada orden individual

  • Compiladores de lenguajes funcionales: convierten código escrito en un lenguaje funcional (como Haskell o Lisp) a lenguaje de máquina. Estos compiladores son especializados en lenguajes con una sintaxis y semántica diferentes a los lenguajes imperativos

  • Compiladores de scripts: convierten código escrito en un lenguaje de script (como JavaScript o PHP) a código ejecutable. Estos compiladores son utilizados para crear programas que se ejecutan en entornos web o de interpretación.

COMPONENTES DE UN COMPILADOR

Los componentes de un compilador son:

  1. Analizador léxico
  2. Análisis sintáctico
  3. Análisis semántico
  4. Generación y optimización de código intermedio
  5. Generador de código objeto
  6. Tabla de Símbolos
  7. Gestor de errores
1. ANALIZADOR LÉXICO

Es la primera fase de un compilador, aquí se lee una secuencia de caracteres de izquierda a derecha del programa fuente, reconocer subcadenas que correspondan a símbolos del lenguaje y retornar los tokens correspondientes y sus atributos.
